# Versioning System

This project uses **CMake-based versioning** with semantic versioning (SemVer).

## How It Works

### Single Source of Truth: CMakeLists.txt

Version is defined in **one place only** - the CMakeLists.txt:

```cmake
project(simple_smpp_server
    VERSION 0.1.0
    DESCRIPTION "SMPP Server - Process-per-client microservices"
    HOMEPAGE_URL "https://github.com/rajesh6115/opensmsc"
    LANGUAGES CXX
)
```

### Template File: config.h.in

A template file `SMPPServer/config/config.h.in` contains placeholder variables:

```c
#define VERSION_MAJOR @PROJECT_VERSION_MAJOR@
#define VERSION_MINOR @PROJECT_VERSION_MINOR@
#define VERSION_PATCH @PROJECT_VERSION_PATCH@
#define PROJECT_NAME "@PROJECT_NAME@"
#define PROJECT_VERSION "@PROJECT_VERSION@"
```

### CMake Generation

During build, CMake replaces placeholders:

```cmake
configure_file(
    "${CMAKE_CURRENT_SOURCE_DIR}/config/config.h.in"
    "${CMAKE_CURRENT_BINARY_DIR}/config/config.h"
    @ONLY
)
```

This creates `build/config/config.h` with actual values:

```c
#define VERSION_MAJOR 0
#define VERSION_MINOR 1
#define VERSION_PATCH 0
#define PROJECT_NAME "simple_smpp_server"
#define PROJECT_VERSION "0.1.0"
```

### Usage in Code

Header file `SMPPServer/include/app_info.hpp` provides access:

```cpp
#include <config.h>  // Generated by CMake

namespace smpp {
    class AppInfo {
    public:
        static constexpr const char* version() { return VERSION_STRING; }
        static constexpr int version_major() { return VERSION_MAJOR; }
        static constexpr bool is_prerelease() { return VERSION_MAJOR == 0; }
    };
}
```

Usage in your code:

```cpp
#include "app_info.hpp"

int main() {
    SPDLOG_INFO("Starting {} {}", smpp::AppInfo::name(), smpp::AppInfo::version());
    
    if (smpp::AppInfo::is_prerelease()) {
        SPDLOG_WARN("Running pre-release version!");
    }
    
    smpp::AppInfo::print_info();
}
```

## Updating Version

### Patch Release (0.1.0 → 0.1.1)

```cmake
# Edit: SMPPServer/CMakeLists.txt
project(simple_smpp_server
    VERSION 0.1.1  ← Change only PATCH
    ...
)
```

Then:
```bash
echo "0.1.1" > VERSION
git add -A
git commit -m "[Release] Version 0.1.1 - bug fixes"
git tag -a v0.1.1 -m "Release version 0.1.1"
```

### Minor Release (0.1.x → 0.2.0)

```cmake
# Edit: SMPPServer/CMakeLists.txt
project(simple_smpp_server
    VERSION 0.2.0  ← Change MINOR
    ...
)
```

### Major Release (0.x.x → 1.0.0)

```cmake
# Edit: SMPPServer/CMakeLists.txt
project(simple_smpp_server
    VERSION 1.0.0  ← Change MAJOR
    ...
)
```

## Build Process

When you build:

```bash
mkdir build
cd build
cmake ..
make
```

CMake automatically:
1. ✅ Reads VERSION from CMakeLists.txt
2. ✅ Generates config.h from config.h.in
3. ✅ Compiles app_info.cpp with version info
4. ✅ Links everything together
5. ✅ Binary contains version info at runtime

---

## Accessing Version at Runtime

### From code:

```cpp
#include "app_info.hpp"

std::cout << smpp::AppInfo::version() << std::endl;  // "0.1.0"
std::cout << smpp::AppInfo::full_version() << std::endl;  // "0.1.0 - pre-release"

if (smpp::AppInfo::is_stable()) {
    // Handle stable version
}
```

### From command line:

```bash
./simple_smpp_server --version
```

To implement `--version`, add to main.cpp:

```cpp
int main(int argc, char* argv[]) {
    if (argc > 1 && std::string(argv[1]) == "--version") {
        smpp::AppInfo::print_info();
        return 0;
    }
    // ... rest of main
}
```

## Logging Version

In your logging initialization:

```cpp
int main() {
    // Initialize spdlog
    spdlog::set_level(spdlog::level::info);
    auto logger = spdlog::stdout_color_mt("main");
    
    // Log version info
    SPDLOG_INFO("Starting {} {}", 
        smpp::AppInfo::name(), 
        smpp::AppInfo::full_version());
    
    SPDLOG_DEBUG("Build type: {}", smpp::AppInfo::build_type());
    SPDLOG_DEBUG("Compiler: {}", smpp::AppInfo::compiler());
}
```

## Advantages of This Approach

✅ **Single source of truth** - Version only in CMakeLists.txt  
✅ **No manual sync** - CMake auto-generates config.h  
✅ **No commits needed** - VERSION file in .gitignore  
✅ **Build-time injection** - Version baked into binary  
✅ **Type-safe** - constexpr prevents runtime errors  
✅ **Standard practice** - Used by many C++ projects  
✅ **Easy to automate** - Scripts can parse CMakeLists.txt  
✅ **Zero overhead** - constexpr means no runtime cost

## File Locations

| File | Purpose | Generated? |
|------|---------|-----------|
| `SMPPServer/CMakeLists.txt` | Version source | No (manually edited) |
| `SMPPServer/config/config.h.in` | Template | No (part of source) |
| `build/config/config.h` | Generated header | Yes (CMake output) |
| `SMPPServer/include/app_info.hpp` | API header | No (part of source) |
| `SMPPServer/src/app_info.cpp` | Implementation | No (part of source) |
| `VERSION` | Convenience file | Optional (informational) |

---

## When to Update Version

| Event | Action |
|-------|--------|
| Bug fix or documentation | Patch bump (0.1.0 → 0.1.1) |
| New feature or improvement | Minor bump (0.1.x → 0.2.0) - After MVP |
| Breaking change | Major bump (0.x.x → 1.0.0) |
| Implementation starts | Stay at 0.1.x until MVP |
| MVP working (1st release) | Bump to 1.0.0 |
